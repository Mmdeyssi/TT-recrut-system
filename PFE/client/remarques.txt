{data} VS res.data : 
1️⃣ Using { data } = await axios.post() (Destructuring Approach)

const { data } = await axios.post("/api/job", jobDetails);
🔹 What happens here?

axios.post() returns a response object.
{ data } extracts only the data property from the response.
Now, data contains only the actual response body, not the full response object.
Example Response from API
{
  "success": true,
  "message": "Job added successfully",
  "job": {
    "id": "12345",
    "title": "Frontend Developer"
  }
}

2️⃣ Using res = await axios.post() (Full Response Object)
const res = await axios.post("/api/job", jobDetails);
console.log(res);
🔹 What happens here?

res contains the entire response object, not just data.
🔹 Example Response from API

{
  data: {
    success: true,
    message: "Job added successfully",
    job: {
      id: "12345",
      title: "Frontend Developer"
    }
  },
  status: 200,
  statusText: "OK",
  headers: {...},
  config: {...},
  request: {...}
}
👉 Use { data } = await axios.post() when you only need the response body.
👉 Use res = await axios.post() when you need metadata like status, headers, etc.

🔍 Why Use req.body.extractedText?
Great question! The reason for using req.body.extractedText instead of just extractedText = pdfData.text is to pass the extracted text to the next middleware or controller (applyForJob).
req.body persists across middlewares:
When a request flows through multiple middlewares, req.body allows data to be shared between them.
extractPdfText is a middleware function:
It processes the file, extracts text, and passes the result to the next function.
The next function (applyForJob) can access it easily:
applyForJob does not need to call pdf-parse again, it just reads req.body.extractedText.
